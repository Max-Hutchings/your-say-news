# Feed metrics

| Domain | Metric name | What the metric measures | User value |
| --- | --- | --- | --- |
| Feed — category ranking | Category candidate state counts | Planned candidate, unseen and previously seen post counts by canonical topic. | Shows whether a category can offer users fresh content. |
| Feed — category ranking | Recycled-seen responses | Planned count of responses that must reintroduce previously seen posts after unseen candidates are exhausted. | Reveals categories that need more fresh coverage. |
| Feed — category ranking | Category popularity concentration | Planned share of category exposure captured by the most highly ranked posts. | Reveals feedback loops that repeatedly amplify a small set of stories or viewpoints. |
| Feed — category ranking | Ranked post age | Planned score/age distribution for category results. | Prevents stale stories or permanent winners from dominating discovery. |
| Feed — category ranking | Impression-report outcomes | Planned display-impression report successes and failures by canonical topic or bounded error code. | Protects repeat suppression so users see varied stories. |
| Feed | Feed build duration | Planned end-to-end feed assembly and ranking duration by feed type and outcome. | Keeps story discovery fast. |
| Feed | Feed build failures | Planned failed feed builds by feed type and bounded error code. | Detects when users cannot reach new stories. |
